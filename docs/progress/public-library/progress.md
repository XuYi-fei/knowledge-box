# Public Library Progress

## 当前阶段

- 公开文库与文档详情已可稳定对外使用，当前重点是继续优化阅读体验、公开目录一致性和内容去重后的展示质量。

## 已完成

- 已新增匿名可访问的公开文库页 `/articles`，支持分类栏、多标签筛选、文章卡片列表与公开正文详情。
- 用户侧工作区 header 已接入 `主页 / 文库 / 工具 / 关于作者 / 更新日志` 路由级切换；原“关于”已改为独立的更新日志页 `/log`。
- 已新增匿名可访问的公开“关于作者”页 `/author`：作者资料由管理员维护结构化数据，前端统一渲染为个人主页/简历页，支持基本信息、教育、技能、工作/实习/项目经历与自定义模块。
- 公开文档详情已具备 Markdown 渲染、标题锚点、大纲、高亮定位、返回对话，以及同专栏文章侧栏。
- 已修复 `/documents/:id` 在工作区内无法滚动的问题，并将大纲高亮、点击跳转和初始锚点定位统一切到页面内部滚动容器。
- 已补齐公开文库单卡宽度、左侧筛选栏拉伸/收起、详情元信息区压缩等阅读体验修复。
- 已补齐公开文库筛选栏与正文区的细滚动条样式，避免左侧分类/tag 滚动条过粗影响阅读；文章列表已统一改为单列横向长条卡片，左侧固定展示“标题 + 标签”两行，右侧展示缩略简介，减少不同窗口宽度下的排版波动。
- 公开目录与分类/标签计数已按“分类 + 标题 + 正文指纹”聚合去重，避免底层重复文档对外重复展示。
- 公开文档详情与后台复用的 Markdown 渲染器现已支持代码块右上角一键复制，便于直接复用文档里的代码示例。
- 公开文档详情与后台复用的代码块复制入口已升级为双按钮工具条，支持复制纯代码与复制 Markdown fenced code block。

- 前端：`npm --prefix frontend run build` 可通过，已覆盖公开文档详情代码块复制按钮与统一 Markdown 代码块渲染组件。

## 待继续推进

- 继续优化公开文库长文阅读、筛选交互与详情导航体验。
- 继续打磨公开作者主页的真实内容填充、外部展示样式和移动端阅读体验。

## 关键注意点

- 公开目录展示层不能直接信任底层文档唯一性；若存在不同 `importKey` 但正文相同的公开文档，需按内容指纹聚合去重。
- 作者主页当前走“结构化数据 + Markdown 渲染”方案；若后续继续扩展字段，优先保持结构化可编辑，而不是回退到管理员手写整页 HTML。
- 文档详情的大纲、锚点和高亮必须绑定同一个真实滚动容器，避免外层 `overflow` 影响交互。
- 专栏能力已贯通导入、审核和公开阅读链路；新增相关变更时要同步检查三侧表现。

# 架构文档模板

在 `architecture.md` 中使用以下结构。

## 1. 文档头

- 目的
- 最近更新时间
- 文档范围

## 2. 项目概览

- 系统做什么
- 主要模块
- 高层请求流转

## 3. 仓库结构

- 顶层目录
- 哪些是活跃运行时代码
- 哪些是遗留、生成物或空目录

## 4. 前端架构

- 运行时和主要库
- 应用启动与 Provider
- 路由与鉴权守卫
- 请求层与基址配置
- SSE 处理
- 按 `app`、`features`、`layouts`、`lib`、`styles` 说明代码组织
- 前端到后端的 API 分组

## 5. 后端架构

- 运行时和主要库
- 启动入口与配置绑定
- 分层：controller、service、repository、domain
- 安全与认证流程
- 管理端 API
- 用户聊天与认证 API
- 文档导入与存储

## 6. 配置矩阵

- `application.yml`
- `application-local.yml.example`
- 项目实际使用的 `spring.*` 配置组
- `knowledge-box.*` 配置组及其含义
- 哪些配置是共享默认值，哪些是本地专用

## 7. 大模型与知识检索链路

- 聊天请求入口
- 会话与消息持久化
- ReAct Agent 创建
- Tool 注册
- 检索回退顺序
- SSE 事件下发与恢复流程

## 8. 核心功能链路

- 用户登录与令牌链路
- 用户聊天流式链路
- 会话历史与删除链路
- 管理端文档上传与索引链路
- 管理端模型 / 配置管理链路

## 9. 特殊说明与维护注意事项

- `local` profile 相关要求
- 敏感配置处理规则
- 空目录或遗留目录说明
- 对维护者重要的已知坑点

## 10. 变更记录

- 使用带日期的条目记录本次文档更新了什么、为什么更新
